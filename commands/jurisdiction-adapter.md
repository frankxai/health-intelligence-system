# /jurisdiction-adapter

Create a jurisdiction-aware health records and care-access adapter.

## Input

- Country or countries.
- Region/state/province/autonomous community if relevant.
- Public/private insurance context if known.
- Main care portals or apps.
- Languages needed.
- Cross-border care or travel context.

## Output

Use `templates/jurisdiction-adapter.md` and `templates/insurance-and-care-access-index.md`.

- Record-access route.
- Portal inventory.
- Records-request checklist.
- Insurance/care-access tracker.
- Clinician-summary language notes.
- Emergency/urgent-care routing reminder.
- Source ledger.

## Rules

- Do not give legal certainty.
- Use official or high-quality sources.
- Mark evidence-check date.
- Tell the user to verify local portal and insurer instructions.
- Do not store identifiers in public files.
